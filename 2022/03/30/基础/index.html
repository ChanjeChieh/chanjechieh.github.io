JavaGuide（基础上）

1 基础

1.1 基础概念与常识

1.1.1 Java 的特点：

  1. 简单易学
  2. 面向对象（封装、继承、多态）
  3. 平台无关性（JVM）
  4. 支持多线程
  5. 可靠性
  6. 安全性
  7. 支持网络编程
  8. 编译与解释并存

1.1.2 JVM、JDK、JRE

  • JVM（Java 虚拟机）
    • JVM 是运行 Java 字节码的虚拟机，JVM 有针对不同操作系统的特定实现，以达到 “Write Once，Run Anywhere” 的目的；
    • JVM 有很多版本，公司、组织、个人均可在满足 JVM 规范的情况下，开发属于自己的专属 JVM（我们平常使用的是 HotSpot JVM）。
  • JDK、JRE
    • JDK 是 Java Development Kit 的缩写，是功能齐全的 Java SDK，包含 JRE，能够创建和编译程序；
    • JRE 是 Java 运行时环境，它是运行已编译 Java 程序所需的所有内容的集合，包括 JVM、Java 类库，Java 命令和其他一些基础构件，不能创建新程序

1.1.3 什么是字节码？采用字节码的好处是什么？

  • 在 Java 中，JVM 可以理解的代码就叫做字节码（就是扩展名为 .clas 的文件），字节码面向 JVM，而不面向特定的机器，一定程度解决了传统解释性语言运行效率低的问题，又保留了解释型语言的可移植性。

1.1.4 为什么 Java 是编译与解释共存的语言？

首先区分一下 编译语言 和 解释语言：

  • 编译语言：通过编译器将源代码一次性翻译成可被该平台执行的机器码，一般情况下执行速度较快，开发效率较低，常见：C、C++、Go、Rust
  • 解释语言：通过解释器一句一句地将代码解释为机器代码后执行，解释语言开发效率高，但执行效率比较低，常见：python、JavaScript、PHP

人们为了改善编译语言的效率，发明了 即时编译 技术，缩小了这两种语言间的差距。这种技术混合了两者的优点，它像编译语言一样，先把程序源代码编译成字节码。到执行期时，再将字节码直译之后，逐行执行。常见：Java、LLSM

1.2.4 continue、break 和 return 的区别是什么？

  1. continue：跳出当前的这一次循环，继续下一次循环；
  2. break：指跳出整个循环体，继续执行循环下面的语句
  3. return：用于跳出所在方法，结束该方法的执行

1.2.5 静态方法为什么不能调用非静态成员？

  1. 静态方法是属于类的，在类加载的时候就会分配内存，可通过类名直接访问；而非静态成员属于实例对象，只有在对象实例化之后才存在，需要通过类的实例对象去访问；
  2. 在类的非静态成员不存在的时候，类的静态成员已经存在了，用已经存在的方法调用在内存中还不存在的非静态成员，属于非法操作

注意❗：虽然可以使用 对象名.方法名调用静态方法，但最好用 类型.方法名 的方式

1.2.6 重载和重写的区别

  • 重载
    • 同样的一个方法能够根据输入数据的不同，做出不同的处理；
    • 发生在同一个类中（或者父类和子类之间），方法名必须相同，参数类型不同、个数不同、顺序不同，方法返回值和访问修饰符可以不同。
    • 发生阶段：编译器
  • 重写
    • 当子类继承父类的相同方法，输入数据一样，但要做出有别于父类的响应时就叫重写
    • 方法名、参数列表i必须相同，其余三点限制：
      • 返回值类型必须比父类更小或者相等
      • 抛出的异常范围必须小于等于父类
      • 访问修饰符必须大于等于父类
    • 父类方法的修饰符为private、final、static 时，子类不可以重写该方法，父类的构造方法也不能重写
    • 发生阶段：运行期

1.3 基础数据类型

1.3.1 Java 中的几种基本数据类型

Java 中共有 8 种基本数据类型：

  • 6 种数字类型：
    • 4 种整数类型：byte、short、int、long
    • 2 种浮点类型：float、double
  • 1 种字符类型：char
  • 1 种布尔类型：boolean

其各自所占空间大小：

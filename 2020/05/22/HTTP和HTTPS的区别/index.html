HTTP 和 HTTPS 的区别

2020-05-22 计算机网络 0 评论 字数统计: 719(字) 阅读时长: 2(分)

一、HTTPS

1.1 HTTPS 简介

HTTPS 是以计算机网络安全通信为目的的传输协议，在 HTTP 下加入了 SSL 层，从而具有了保护交换数据隐私，以及完整性，提供对网站服务器身份认证的功能，或者说是 安全版的 HTTP。

1.2 SSL 简介

SSL 的全称是 Security Sockets Layer，安全套接层，是一种为网络通信提供安全及数据完整性的一种安全协议，位于 TCP 和各 应用层 之间，是操作系统对外提供的 API，SSL3.0 版本之后更名为 TLS，采用 身份验证 和 数据加密 保证网络通信的安全和数据的完整性。

1.2.1 加密的方法

  • 对称加密：加密和解密都是用同一个密钥
  • 非对称加密：加密和解密使用不同密钥（即 公钥 和 私钥）
  • 哈希算法：将任意长度的信息转换为固定长度的值（MD5）
  • 数字签名：在一段数据后加一段哈希后的值

1.2.2 HTTPS 数据传输流程

在进行数据传输前，网站服务器会和 WEB 浏览器进行一次握手，在握手时验证加密信息。

  1. 浏览器将支持的加密算法信息发给服务器
  2. 服务器根据浏览器所支持的加密方式在自己支持的加密方式中选出一套加密算法和哈希算法，将要验证的信息以证书的形式发给 WEB 浏览器
  3. 浏览器收到证书后会验证证书的合法性，若信任，WEB 浏览器会随机生成一段密码，并使用证书中的公钥加密，再使用约定好的哈希算法握手，并生成随机数对消息进行加密
  4. 服务器收到网站发送的数据后，会使用网站本身的私钥将信息解密确定密码，再使用这个密码解密握手信息，并验证哈希是否与 WEB 浏览器一致，再使用密码加密新的信息发给浏览器
  5. 客户端浏览器解密并计算经密码加密的握手信息，若与服务器发送的哈希值一致，随便服务器和浏览器会使用之前浏览器之前生成的随机密码和对称加密算法进行加密并交换数据

二、HTTP 和 HTTPS 的区别

如下表：

HTTP HTTPS
不需要 需要到 CA 申请证书（花费）
明文传输 密文传输（使用 SSL 加密传输协议）
80 端口 443 端口
无状态 HTTP+加密+认证+完整性保护

看起来 HTTPS 更安全，但实际上人们习惯在访问某个页面时直接在浏览器中直接输入网站域名，这时，浏览器会默认填充 http://

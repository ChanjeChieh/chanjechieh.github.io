HTTP 简介

2020-05-21 计算机网络 0 评论 字数统计: 1.4k(字) 阅读时长: 4(分)

一、HTTP 是什么

超文本传输协议 HTTP 是一个基于请求和响应模式的、无状态的、应用层的协议，常基于 TCP 的连接方式，绝大多数的 web 开发都是构建在 HTTP 协议上的 web 应用。

二、HTTP 的特点

  1. 支持客户/服务器模式（C/S模式）
  2. 简单快速：客户端向服务器发送请求时只需要给出路径和请求方法（GET、POST等）即可
  3. 灵活：可以传输任意类型的数据，其类型由 content-type 加以标记
  4. 无连接：每次连接只处理一个请求（节省传输时间）
  5. 无状态：协议对于事务处理没有记忆能力

三、HTTP 请求报文结构

如下图，HTTP 请求报文主要由：

  1. 请求行：由请求方法、url、协议版本构成
  2. 请求头部：由键值对构成，这些报头用来设置一些参数
  3. 空行：即使 请求正文 为空也要有这个空行
  4. 请求正文（数据）：只在 POST 请求方法中用到

五、请求 / 响应的步骤

  1. 客户端连接到 Web 服务器：一个 HTTP 客户端通常是浏览器，与 Web 服务器的 80 端口建立一个套接字连接
  2. 发送 HTTP 请求：即通过 TCP 套接字向 Web 服务器发送一个文本的请求报文
  3. 服务器接受请求并返回 HTTP 响应：Web 服务器解析该请求，并请求资源，服务器将资源副本写到 TCP 套接字中，由客户端读取
  4. 释放连接 TCP 连接
  5. 客户端浏览器解析 HTTP 内容：首先查看状态行，查看请求是否成功，然后解析每一个响应头，最后根据 HTML 的语法将页面在浏览器中显示出来

六、相关问题

1、在浏览器地址栏中输入带有 HTTP 前缀的 URL，按下回车后经历的流程：

  1. 浏览器会根据 URL 逐层查询 DNS 服务器缓存，解析 URL 中的域名所对应的 IP 地址，DNS 从近到远依次是 浏览器缓存、系统缓存、路由器缓存、IPS 服务器缓存、根域名服务器缓存、顶级域名服务器缓存，若在这个过程中找到了则直接返回，不再查询后面的缓存
  2. TCP 连接：找到 IP 地址后会根据该 IP 地址和对应端口（默认80）和服务器建立 PCP 连接
  3. 浏览器发送读取文件的 HTTP 请求，该请求将发送给服务器
  4. 服务器对浏览器请求做出相应，并将带有数据的 HTTP 响应报文发送给浏览器
  5. 浏览器收到 HTML 后渲染页面
  6. 浏览器释放 TCP 连接，连接结束

2、常见的 HTTP 状态码

状态码由三位数组成，第一个数字定义了响应的类别，主要有五种类型：

  • 1xx：指示信息–表示请求已经接收，并继续处理
  • 2xx：成功–表示请求已被成功接收、理解、并接受
  • 3xx：重定向–要完成请求必须进行更进一步的操作
  • 4xx：客户端错误–请求有语法错误或请求无法实现
  • 5xx：服务端错误–服务端未能实现合法的请求
  1. 200 OK：正常返回信息
  2. 400 Bad Request：客户端请求有语法错误，不能被服务器所理解
  3. 401 Unauthorized：请求未授权，这个状态代码必须和 WWW-Authorized 报文域一起使用，表示客户端没有权限请求某些资源
  4. 403 Forbidden：服务器收到请求，但拒绝提供服务（IP 被封锁）
  5. 404 Not Found：请求资源不存在
  6. 500 Internet Server Error：服务器发生了不可预期的错误
  7. 503 Server Unavailable：服务器当前不能处理客户端的请求，一段时间后可能恢复正常

3、GET 请求和 POST 请求的区别

三个方面：

  1. 从HTTP报文上来看：
    GET 请求方式将请求信息放在 URL 后，请求信息和 URL 之间以 ? 隔开，请求信息的格式为键值对，受 URL 长度限制；
    POST 请求方式将请求信息放在报文体中，想获得请求信息的话，必须解析报文，因此安全性较 GET 方式高，但由于获得报文中的信息比较容易，所以实际上安全性两者没有太大区别。具体的安全性问题需要交给 HTTPS 来解决
  2. 从数据库层面上看：
    GET 请求方式因为是做查询用，所以符合 幂等性 和 安全性
    POST 不符合

幂等性：幂等性是指对数据库的一次操作和多次操作获得的结果是一致的，则认为符合幂等性
安全性：对数据库的操作没有改变数据，则认为符合安全性

  1. 从其他层面上看：
    GET 请求方式可以被缓存、存储，大部分的 GET 请求都会被 CDN 缓存
    POST 不行，由于非幂等且不安全，所以其必须交给 Web 服务器处理

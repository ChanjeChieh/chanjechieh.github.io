Cookie 和 Session 的区别

2020-05-21 计算机网络 0 评论 字数统计: 744(字) 阅读时长: 2(分)

因为 HTTP 是无状态的，所以我们每次访问有登录需求的页面都要输入账号密码。Cookie和

一、Cookie

1.1 Cookie 简介

Cookie 是客户端的解决方案，Cookie 是由服务器发送给客户端的特殊信息，而这些信息以文本文件的形式存放在客户端，客户端每次向服务端发送请求时都会带上这些信息，用户就不用再填写用户名和密码了。

或者说：

当用户访问一个支持 Cookie 的网站时，用户会提供用户名、密码等个人信息给服务器，而服务器在向客户端发送超文本的同时，会将这些信息放在 Response-Header 中，这时，客户端会将这些信息统一存在一个位置，以后客户端再发送请求会将这些信息（Cookie）放在请求头一并发送。

1.2 Cookie 的设置和发送过程

首先客户端发送一个 HTTP 请求到服务端，然后服务端发送一个包含 Set-Cookie 头部的 HTTP 响应到客户端，其中就，客户端再发送一个包含 Cookie 头部的 HTTP 请求到服务端，此时 服务端再发送一个 HTTP 响应到客户端。

二、Session

2.1 Session 简介

Session 机制是一种在服务端的机制，服务器采用一种类似于散列表的结构保存信息，当程序需要为某个客户端的请求创建一个 session 的时候，服务器首先检查这个请求里是否已经包含一个 session 标识（session id，是一个既不重复也不容易被找到生成规律的序号）。如果已经包含，则证明以前创建过 session，服务端便使用这个 session id 到散列表中检索出相应的 session 来；如果不包含，就为这个客户端创建一个 session，并生成一个相应的 session id，

2.2 Session 的两种实现方式

  1. 使用 Cookie 实现：
    服务器给每个 Session 分配一个唯一的 JSESSIONID 并通过 Cookie 发送给客户端，当客户端发起新的请求时，将在新的请求头中携带这个 id，这样，服务器就能找到与这个客户端相应的 Session

  2. 使用 URL 回写实现：
    服务器在发送给浏览器页面的所有页面中都携带 JSESSIONID 的参数，这样，使用浏览器点击任意链接都会把 JSESSIONID 一同发送

不管使用哪种方式，都需要 JSEESIONID。

三、Cookie 和 Session 的区别

  1. Cookie 数据存放在客户的浏览器上，SESSION 数据存放在服务器上
  2. Session 较 Cookie 更为安全：人们可以分析存放在本地的 Cookie 并进行 Cookie 欺骗
  3. Session 会在一定时间内保存在服务器上，当页面访问量较多时会影响服务器性能，若考虑到服务器性能，应使用 Cookie

volatile 关键字

2020-05-23 Java 0 评论 字数统计: 485(字) 阅读时长: 1(分)

volatile 的作用有：

一、保证变量的可见性

volatile 保证了不同线程对其所修饰变量进行操作时的可见性，即当一个线程修改该变量时，这个新值对其他线程来说是立即可见的。

  1. 使用 volatile 关键字会强制将修改的值写入主存
  2. 使用 volatile 关键字的话，当线程 B 进行修改时，会导致线程 A 的工作内存中缓存变量的缓存行无效
  3. 由于线程 A 的工作内存中缓存变量的缓存行无效，所以线程 A 再次读取变量的值时就回去主存中读取，那么在修改 stop 值时（这里包括两个操作：修改线程 B 工作内存中的值，然后将修改后的值写入内存），会使得线程 A 的工作内存中缓存变量 stop 的缓存行无效，线程 A 读取时发现自己的缓存行无效，便等待缓存行对应的地址被更新之后去对应的主存读取最新的呃值。

二、保证有序性（禁止指令重排序）

为什么在单例模式的双检锁实现方法中对 变量 使用了 volatile 关键字进行修饰？

因为 instance = new DCL(); 这条语句中，其并非一个原子操作，在 JVM 中这条语句大概做了下面三件事：

  1. 给 instance 分配内存；
  2. 调用 DCL 的构造函数来初始化成员变量
  3. 将 instance 对象指向分配的内存空间

但由于存在指令重排序的问题，第二步和第三步可能颠倒执行，这种情况下，当第一个线程执行到 3 但是没有执行 2 时，第二个进程过来，判断这个 instance 不是 null，然后直接返回 instance，便产生了错误，而使用 volatile 关键字就可以避免这种错误发生。
